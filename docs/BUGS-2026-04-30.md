## Bug Report — UI/UX Issues on Roll Page + Cook Tab Error

**Reported:** 2026-04-30 (evening)
**Status:** Open, queued for Codex T005 or Phase 16 hotfix
**Priority:** High (blocks core user flow)

---

### Bug 1: Portion Size Not Visible on Roll Result

**Where:** `/` (Roll a Meal screen) after a successful roll
**Expected:** The rolled meal result should clearly show the portion count (e.g., "2 portions", "4 portions") somewhere prominent.
**Actual:** Portion size is hidden. The user can't see how many servings the recipe yields without navigating away.

**Related code:**
- `web/components/roll-page.tsx` line ~247: `portions: controls.portions` — sent via API
- The response likely includes portions but it's not rendered in the result card.

**Suggested fix:** Add a portion display in the rolled result view, likely near the price/chip area or as a dedicated badge.

---

### Bug 2: No "I Want to Cook This" Action After Rolling

**Where:** `/` (Roll a Meal screen) after a successful roll
**Expected:** After rolling a meal, there should be a clear primary CTA like **"Cook this meal"** or **"I want to cook this"**.
**Actual:** The roll result only shows the component list. No action button appears below it. The user must manually navigate to the Cook tab or find another way to start the flow.

**Related code:**
- `web/components/roll-page.tsx` line ~328: `goCook` callback exists but is only triggered by a small button at line ~758 (`<Button type="button" size="sm" onClick={goCook}>`).
- The button is likely too small or poorly placed. It should be the primary action after a roll.

**Suggested fix:**
1. Replace the small `size="sm"` button with a **large primary CTA** (`size="lg"` or full-width) positioned directly under the rolled meal result.
2. Button text should read: **"Cook this meal"** or **"Start cooking"**.
3. Ensure the button navigates to `/recipe` and triggers the `goCook` flow.

---

### Bug 3: Error When Navigating to Cook Tab After Rolling

**Where:** `/cook` page, after coming from a rolled meal
**Expected:** The Cook tab should show the recipe/cooking view for the most recently rolled meal.
**Actual:** A database error occurs when navigating to the Cook tab after rolling a meal. User suspects it's a database query error ("something not found" — possibly a missing `bowl_id`, `recipe_id`, or `component_id` when the cook page tries to load the rolled data from storage/session state).

**How to reproduce:**
1. Go to Roll tab
2. Roll a meal
3. Attempt to navigate to the Cook tab (either via bottom nav or via the existing small Cook button)
4. Error page appears

**Suspected cause:** `web/components/cook-page.tsx` tries to read `ROLLED_BOWL_STORAGE_KEY` from `sessionStorage` or local state. If the rolled bowl was never persisted correctly, or the cook page expects a different data shape, it fails.

**Suggested fix:**
1. Guard against missing sessionStorage data gracefully (show fallback message).
2. Verify that `goCook` in `roll-page.tsx` actually writes to the expected storage key before navigating.
3. Check `cook-page.tsx` session state loading logic for null/undefined handling.

---
