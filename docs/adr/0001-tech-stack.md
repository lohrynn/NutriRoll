# 0001 — Tech stack

- **Status:** Accepted
- **Date:** 2026-04-22
- **Decision owner:** NutriRoll core
- **Source of truth:** [`PROJECT_VISION.md`](../../PROJECT_VISION.md) § "Tech stack"

## Context

NutriRoll is a mobile-first installable PWA that recommends bowl-style meals.
We need a stack that is small, productive for a tiny team, hostable cheaply,
and able to satisfy the explicit constraints in `PROJECT_VISION.md`:
mobile-first PWA on iOS 16.4+ / Android Chrome 120+, online-by-default with
offline-as-progressive-enhancement, single relational store, BYO-key LLM,
strict security baseline, and explainable recommendations from a
framework-free domain layer.

## Decision

Adopt the stack documented in `PROJECT_VISION.md` § "Tech stack" verbatim:

- **Frontend:** Next.js 15 (App Router, RSC) on React 19, TypeScript strict,
  Tailwind CSS v4 (CSS-first config), shadcn/ui, TanStack Query for server
  state, Zustand for client state, Serwist for PWA / service worker /
  background-sync, next-intl (en + de) for i18n.
- **Backend:** FastAPI + Pydantic v2 + uvicorn on Python 3.13 managed by
  `uv`. PostgreSQL 17 as the only datastore via SQLAlchemy 2.0 (async) +
  Alembic. APScheduler for in-process scheduled jobs. Domain layer in
  `server/nutriroll/domain/` is framework-free.
- **API contract:** REST + OpenAPI; the typed TS client is generated by
  `openapi-typescript` + `openapi-fetch`. URL-prefixed versioning (`/v1/`).
  Errors as RFC 7807.
- **Auth:** Anonymous device-token (`X-Device-Id`) + HMAC session cookie.
  No accounts in v1.
- **LLM:** BYO-key, envelope-encrypted at rest, server-proxied, per-feature
  opt-in, deterministic fallback mandatory.
- **Hosting:** Vercel (web) + Fly.io app + Fly Postgres (server).
- **Observability:** structlog JSON + Fly Prometheus + Sentry.
- **Testing:** pytest + httpx + hypothesis + testcontainers (server);
  Vitest + RTL + MSW (web); single Playwright happy-path e2e.
- **Tooling:** pnpm workspace + uv, Biome (web), ruff + pyright strict
  (server), lefthook pre-commit, GitHub Actions CI, Vercel + flyctl deploy.

## Consequences

- **Positive:** Single deployable per side. Type safety end-to-end without
  hand-written API types. Domain layer is testable without infra. PWA path
  avoids app stores entirely.
- **Negative / accepted trade-offs:** No native APIs (push, background
  geofencing) without future SW/Web Push work. iOS 16.3 and below are
  unsupported by design. No staging environment in v1 — preview deployments
  cover that need.
- **Reversal cost:** Swapping any pillar (Next.js, FastAPI, Postgres) would
  require a new ADR and explicit human approval. Auxiliary choices (Biome,
  shadcn/ui, Serwist) are easier to revisit.

## Out of scope (v1)

Per the vision: no DuckDB, no Redis, no message broker, no native apps,
no third-party analytics, no real CRDT sync, no collaborative filtering.
